# 心镜(MindMirror)—— 心理状态监测智能体

## 一、项目概述

**心镜**是一个部署在智慧教室环境中的 AI 智能体系统——通过摄像头阵列自动采集学生面部视频，调用多模态 AI 模型实时分析情绪状态，并在每日晚间进行深度心理健康评估，生成分级预警，辅助班主任和心理老师及时介入。

一句话理解：**教室里的"情绪CT机"——每15分钟扫一次得到短时情绪，每晚10点出一份心理报告。**

## 三、核心工作流程：双环状态机

这是整个系统最核心的设计。智能体的工作被建模为两个独立的循环：

### 3.1 内环：实时情绪感知（每15分钟触发一次）

```
教室摄像头开启摄像 → 多模态情绪识别 → 华为云存储
```

三个步骤依次执行，**全程不调用大模型**（纯确定性流水线），单次耗时 <1.5秒：

| 步骤 | 做什么 | 谁来做 |
|------|--------|--------|
| **采集(collect)** | 拿到视频片段路径（手动上传的视频片段，或定时任务用的默认摄像头路径） | collect_node |
| **识别(recognize)** | 调用多模态情绪识别工具，返回面部情绪+前庭振动K系数+融合结果 | EmotionRecognitionTool |
| **存储(store)** | 结果写入本地SQLite + 打包上传至云端OBS | store_node |



### 3.2 外环：心理健康评估（每晚22:00触发一次）

```
拉取全天情绪数据 → 时序分析 → 分级预警 → 多渠道通知
```

四个步骤依次执行：

| 步骤 | 做什么 | 谁来做 |
|------|--------|--------|
| **聚合(aggregate)** | 从数据库拉取每位学生当天的所有情绪记录，建成时序特征矩阵 | aggregate_node |
| **分析(analyze)** | 两层混合架构：Layer 1 纯数学计算5项指标（熵/趋势/恢复速度/积极占比/突变），Layer 2 调用**智谱AI GLM-4-Flash** 推理剩余7项指标（稳定性/负面累积/社交/唤醒度/睡眠/压力/综合评分），并生成风险评估与建议。LLM不可用时自动降级为规则引擎。 | MentalHealthAnalysisTool + 智谱AI |
| **预警(alert)** | 按关注度三档分级：🟢 绿色(正常) 🟡 黄色(关注) 🔴 红色(干预) | alert_node |
| **反馈(feedback)** | 绿色仅看板通知，黄色加微信，红色加邮件+短信并自动创建干预工单 | MultiChannelFeedbackTool |

==后面我们应该单独训练出一个心理专业的大模型==

### 3.3 一张图理解双环

```
时间轴 ──────────────────────────────────────────────────→
  08:00        10:00        12:00        ...        22:00
   │            │            │                        │
   🔄            🔄            🔄                      🌙
  内环          内环          内环                    外环
 采集识别      采集识别      采集识别                聚合分析
  存储          存储          存储                   预警反馈
```

## 四、技术栈

| 层级 | 技术 | 为什么选它 |
|------|------|-----------|
| 后端框架 | FastAPI (Python) | 原生支持异步、自动生成 Swagger 文档、SSE 流式响应简单 |
| AI 编排 | LangGraph | 状态机建模能力强，支持 astream 流式输出，天然适合 ReAct 模式 |
| 大模型 | DeepSeek V4 + 智谱AI GLM-4-Flash | DeepSeek 用于 ReAct 智能体推理链，GLM-4-Flash 用于时序心理指标推理与反馈通知生成。均兼容 OpenAI 接口 |
| 数据库 | SQLite | 零配置，文件级便携，数据量不大时完全够用，可平滑迁移 PostgreSQL |
| 定时任务 | APScheduler | 轻量，支持 interval(15min) + cron(22:00) 两种触发模式 |
| 前端 | Vue 3 + Vite + Pinia | 响应式数据绑定、组件化开发、Vite 热更新快 |
| 图表 | ECharts | 功能全面，折线图+饼图开箱即用 |
| 流式通信 | SSE (Server-Sent Events) | 单向推送，比 WebSocket 简单，浏览器原生支持 EventSource |

### 6.1 多模态情绪识别工具 (EmotionRecognitionTool)

```
输入: image_path + student_id + baseline_mood
输出: 面部情绪 + 前庭情绪 + 融合情绪 + 12项质量指标
```

**核心功能**：
- 集成面部微表情API与前庭振动API双模态识别
- 加权融合策略（面部0.6 + 前庭0.4）生成综合情绪评估
- 置信度差异校验：差异>35%时自动触发复核机制
- 偏差检测：与历史基线对比，偏离>30%时发出预警
- 目标准确率：>92%

**情绪分类**：
| 情绪 | Valence | Arousal | 说明 |
|------|---------|---------|------|
| 开心 | +0.8 | +0.6 | 正效价、高唤醒 |
| 平静 | +0.3 | -0.3 | 中性、略低唤醒 |
| 悲伤 | -0.7 | -0.2 | 负效价、低唤醒 |
| 焦虑 | -0.4 | +0.7 | 负效价、高唤醒 |
| 愤怒 | -0.6 | +0.8 | 负效价、高唤醒 |
| 惊讶 | +0.2 | +0.9 | 中性、最高唤醒 |

### 6.3 时序心理健康分析工具 (MentalHealthAnalysisTool)

```
输入: student_id + 7天records + baseline
输出: 12项指标 + LLM推理链 + 风险等级 + 建议 + 模式识别 + 趋势预测
```

**两层混合架构**：

```
Layer 1: 纯数学计算（5项，无可争议）     Layer 2: 智谱AI GLM-4-Flash 推理（7项 + 深度分析）
┌───────────────────────────────┐      ┌────────────────────────────────────┐
│ ② 情绪波动熵值 (Shannon熵)    │      │ ① 情绪稳定性指数                    │
│ ⑤ 日间情绪趋势 (线性回归)     │      │ ③ 负面情绪累积度                    │
│ ⑦ 情绪恢复速度 (转正频率)     │      │ ④ 社交互动频次                      │
│ ⑩ 积极情绪占比 (正面/总数)    │      │ ⑥ 唤醒度异常指数                    │
│ ⑪ 情绪突变检测 (>30%变化)     │      │ ⑧ 睡眠质量预测                      │
└───────────────┬───────────────┘      │ ⑨ 压力累积指数                      │
                │                      │ ⑫ 综合心理健康评分                   │
                │                      ├─ 风险评估（绿/黄/红 + 推理链）       │
                │ 作为事实数据喂入 →    ├─ 干预建议（按优先级 + 目标角色）      │
                │                      ├─ 风险因素 + 保护因素识别             │
                │                      ├─ 行为模式检测（pattern_name/confidence）│
                │                      └─ 趋势预测（forecast + key_uncertainty） │
                │                      └────────────────────────────────────┘
                │                              │
                │                    LLM不可用时自动降级
                │                              │
                │                    ┌─────────▼──────────────┐
                │                    │ RuleEngine-Fallback    │
                │                    │ (LSTM-Transformer 模拟 │
                │                    │  + 规则引擎兜底)       │
                │                    └────────────────────────┘
                │                              │
                └──────────────┬───────────────┘
                               ▼
                     12项指标完整输出 + 风险等级 + 建议
```

**12项心理健康指标分层明细**：

| # | 指标 | 计算层 | 计算方式 |
|---|------|--------|---------|
| ① | 情绪稳定性指数 | Layer 2 (LLM) | 综合熵/方差/突变次数推理 |
| ② | 情绪波动熵值 | Layer 1 (数学) | Shannon熵 + 5区间离散化 |
| ③ | 负面情绪累积度 | Layer 2 (LLM) | 综合负面占比/趋势/分布推理 |
| ④ | 社交互动频次 | Layer 2 (LLM) | 基于记录密度 + 时间分布推理 |
| ⑤ | 日间情绪趋势 | Layer 1 (数学) | 线性回归斜率 → 改善中/稳定/下降中 |
| ⑥ | 唤醒度异常指数 | Layer 2 (LLM) | 基于 arousal 时序波动推理 |
| ⑦ | 情绪恢复速度 | Layer 1 (数学) | 负面→正面转换频次 + 计算平均恢复间隔 |
| ⑧ | 睡眠质量预测 | Layer 2 (LLM) | 基于晚间情绪 + arousal 模式推理 |
| ⑨ | 压力累积指数 | Layer 2 (LLM) | 综合负面累积/恢复速度/趋势推理 |
| ⑩ | 积极情绪占比 | Layer 1 (数学) | 正面情绪记录 / 总记录数 |
| ⑪ | 情绪突变检测 | Layer 1 (数学) | 相邻分数变化 > 30% 计数 |
| ⑫ | 综合心理健康评分 | Layer 2 (LLM) | 加权综合所有指标 + 专业判断 |

**LLM 推理详情**：
- 模型：智谱AI GLM-4-Flash（temperature=0.3, max_tokens=2048, JSON强制输出）
- 输入：Layer 1 的5项数学事实 + 情绪分布 + valence/arousal 序列摘要
- 输出：7项指标值 + 风险评估（等级/原因/推理链）+ 建议 + 模式识别 + 趋势预测
- System Prompt 设定为"学校心理健康评估专家"角色，要求紧扣数据推理、不编造症状

**降级容错**：
- LLM 调用成功 → model_version = "ZhipuGLM-4-Flash-v1.0"
- LLM 调用失败（网络/API异常/JSON解析失败）→ 自动回退 rule-engine
- 回退时由 `_lstm_transformer_analysis()` (模拟) + `_determine_risk_level()` (规则引擎) + `_generate_suggestions()` (模板) 完整产出，model_version = "RuleEngine-Fallback-v2.1"
- 无论 LLM 可用与否，12项指标始终完整输出，不阻断下游流程

**风险等级判定**：
- 🟢 绿色：综合评分≥0.7，无红色指标
- 🟡 黄色：综合评分0.4-0.7，或有2项以上黄色指标
- 🔴 红色：综合评分<0.4，或有3项以上黄色指标，或检测到风险模式

### 6.4 多渠道反馈工具 (MultiChannelFeedbackTool)


### SSE 流式事件类型

```
event: thought      → 智能体当前想法："开始分析张三的情绪数据..."
event: action       → 调用了哪个工具："多模态情绪识别"
event: observation  → 工具返回了什么："开心, 置信度0.88"
event: final        → 最终结论："张三今日情绪状态良好"
```

## 十一、关键设计决策

| 决策 | 做法 | 原因 |
|------|------|------|
| 内环不用大模型 | 直接调工具 | 采集→识别是确定性流水线，不需要推理，延迟 <1.5秒 |
| 外环用双大模型 | DeepSeek V4 (ReAct推理) + 智谱AI GLM-4-Flash (指标推理+通知) | 时序心理分析和通知生成需上下文理解和多步推理，大模型擅长；指标计算分层（数学+AI）兼顾可控性与专业性 |
| 双环分离 | 两张独立的 LangGraph 图 | 各自独立触发、独立执行、互不影响 |
| 工具用 stub 先跑通 | 返回模拟数据 | 前后端可以并行开发，接口定好了随时替换 |
| SQLite 而非 PostgreSQL | 文件数据库 | 零运维成本，数据量小的场景足够，后期可迁移 |
| SSE 而非 WebSocket | 单向推送 | Agent 输出只需要服务器→前端单向推送，SSE 更简单 |

## 十二、团队 ==工具接入指南==

### 如果你是工具开发者

你需要替换的文件在 `backend/tools/` 下，每个文件里有一个类：

1. **`emotion_recognition.py`** → 把 `execute()` 里的随机数换成真实的多模态融合逻辑
2. **`obs_storage.py`** → 把本地 JSON 写入换成华为云 OBS SDK 调用
3. **`mental_health.py`** → 已接入智谱AI GLM-4-Flash推理7项指标，替换`_calculate_indicators()`中的规则引擎兜底逻辑或升级`_llm_deep_analysis()`中的 prompt 即可
4. **`feedback.py`** → 把 mock 消息换成真实的微信/邮件/短信发送

**替换时的规则：**
- 保持 `name` 属性和 `execute()` 的参数签名不变
- `execute()` 必须返回 `ToolResult(success=bool, data=dict, error=str)`
- 替换后运行 `python tests/test_prompt.py` 确认12个测试通过

### 如果你要改 System Prompt

文件在 `prompts/system_prompt.py`。Prompt 里定义了智能体的"人格"和工具使用规则。改完后运行 `python tests/test_prompt.py` 确认解析器还能正确解析。

### 如果你要加新的 API 端点

1. 在 `backend/api/routes/` 下新建或修改路由文件
2. 在 `backend/main.py` 中注册路由
3. 如果需要新的数据结构，在 `backend/schemas/` 下添加 Pydantic 模型
4. 前端调用在 `frontend/src/api/index.js` 中添加对应函数
